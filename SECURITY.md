# Security policy

## Supported versions

Only the latest commit on the default branch is supported.

## Reporting a vulnerability

Do not open a public issue containing credentials, node URLs, subscription
content, server IPs, logs with sensitive values, or private keys.

Before the first public release, enable GitHub private vulnerability reporting
for this repository. Use that private reporting channel for security issues.
If it is unavailable, contact the maintainer through their GitHub profile and
share only the minimum redacted information needed to reproduce the issue.

Rotate any credentials that may have been exposed before reporting the issue.
